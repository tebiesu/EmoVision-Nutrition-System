# Task 01：项目定位与目标

## 目标
- 明确系统定位：个人饮食健康智能助手（情感识别 + 视觉分析 + 建议 + 趋势）。
- 固化工程优先路线：`Spring Boot + Vue3 + MySQL`。
- 固化 AI 通道：主 `OpenAI/Gemini(New API)`，备 `SiliconFlow`。

## 非目标（必须约束）
- 不做自训练大模型与在线持续学习。
- 不做多角色复杂权限系统（单用户/轻账号）。
- 不做医疗诊断，仅提供健康管理建议。

## 执行步骤
1. 输出《需求边界说明》并写入仓库文档。
2. 定义“必须有”和“可以后置”功能清单。
3. 评审后冻结范围，作为后续任务输入。

## 产出物
- 需求边界文档（含目标、非目标、验收口径）。

## 验收标准
- 目标/非目标无冲突。
- 所有后续任务均引用本边界，不再扩张范围。
