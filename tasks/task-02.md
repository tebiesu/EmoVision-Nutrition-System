# Task 02：技术栈与版本基线

## 目标
- 建立可复现、可部署的技术基线与版本约束。

## 技术栈清单
- 后端：JDK 17、Spring Boot 3.3.x、Spring Web、Validation、Security(JWT)、MyBatis-Plus 3.5.x、MySQL 8.0+、Maven 3.9+
- 前端：Node.js 20+、Vue3、Vite5、Element Plus、Axios、ECharts
- 支撑：MapStruct（可选）、SpringDoc OpenAPI、Logback + MDC

## 执行步骤
1. 建立版本约束表（后端/前端/数据库/构建工具）。
2. 编写环境准备说明（本地/服务器）。
3. 设置统一编码与依赖管理策略。

## 产出物
- 技术基线文档
- 环境检查清单

## 验收标准
- 新成员按文档能在同日完成环境启动。
- 无“版本漂移”导致的构建失败。
