# Task 10：日志、监控与异常

## 目标
- 建立可追踪、可定位问题的运维能力。

## 异常码
- `A0000` 成功
- `A0400` 参数错误
- `A0401` 未认证
- `A0500` 系统异常
- `P1001` Provider 超时
- `P1002` Provider 全部失败

## 日志策略
- 请求生成 `traceId`
- Provider 调用写结构化日志与调用日志表
- `token/key` 等敏感信息脱敏

## 执行步骤
1. 统一异常处理器实现。
2. 接口日志与 Provider 日志埋点。
3. 关键链路的 trace 串联验证。

## 验收标准
- 任意失败请求可通过 traceId 追到根因。
- 线上日志不出现明文敏感数据。
