# MVP 功能优先级清单

## 分级标准
- P0：答辩与闭环必须能力，首版必须完成。
- P1：增强能力，可在 P0 稳定后实现。
- P2：优化能力，延期不影响主线交付。

## P0（必须有）
1. 用户档案管理（目标、禁忌、活动水平）
2. 餐食上传与识别（图片输入 + 食物识别 + 营养估算）
3. 情感识别（图像/文本输入，输出标签与置信度）
4. 规则建议引擎（基于情绪 + 营养 + 目标 + 禁忌）
5. 基础看板（概览指标、7/30 天趋势）
6. 主备 Provider 路由与失败回退（含调用追踪）

## P1（可后置）
1. 规则配置可视化（规则管理界面）
2. 更多建议模板与解释增强
3. 监控面板增强（错误分类、延迟分布）

## P2（可延期）
1. 多终端深度适配
2. 高级报表导出（PDF/Excel）
3. 更复杂的个性化模型与实验性策略

## 冻结原则
1. 当前迭代仅允许 P0 进入开发与验收。
2. P1/P2 不得阻塞 P0 交付。
3. 新需求默认进入 P1，需评审后才可提升优先级。
