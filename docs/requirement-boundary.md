# 需求边界说明（Task 01）

## 1. 文档目的
本文件用于冻结项目范围，作为后续任务（Task 02-15）的统一输入，避免需求膨胀。

## 2. 项目目标（In Scope）
1. 构建一个可演示、可交付的 Web 系统，形成完整闭环：
   - 饮食记录
   - 情感识别
   - 个性化建议
   - 趋势分析
2. 固化工程主路线：
   - 后端：Spring Boot
   - 前端：Vue 3
   - 数据库：MySQL
3. 固化 AI 通道策略：
   - 主通道：OpenAI/Gemini（New API）
   - 备通道：SiliconFlow

## 3. 非目标（Out of Scope）
1. 不进行自训练大模型与在线持续学习。
2. 不实现多角色复杂权限系统，仅支持单用户/轻账号场景。
3. 不提供医疗诊断能力，仅输出健康管理建议。

## 4. 功能边界
### 4.1 必须具备（MVP/P0）
1. 用户档案管理
2. 餐食上传与识别
3. 情感识别
4. 规则建议生成
5. 基础看板（概览与趋势）

### 4.2 可以后置（P1/P2）
1. 复杂规则可视化配置
2. 多终端适配（移动端深度优化、跨端增强）
3. 高级报表导出

## 5. 验收口径
1. 业务闭环可端到端跑通：上传餐食 -> 识别 -> 情感分析 -> 建议 -> 看板展示。
2. 主通道失败时可触发备通道回退，且可追踪调用记录。
3. 所有后续设计与实现均不得突破本文件定义的边界。

## 6. 变更控制
1. 如需新增范围，必须记录变更原因、影响模块、收益与风险。
2. 变更评审通过前，不进入开发与测试阶段。
